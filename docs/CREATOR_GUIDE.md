# SkyOS Creator Guide

SkyOS is built to help creators move from idea to usable output without losing trust, structure, or identity. Creator workflows should feel fast, but not careless.

## Creator Workflow

A healthy SkyOS creator loop:

1. Start from Home to see current signals.
2. Use AI or Agent to shape the idea.
3. Move into Music, Video, Shop, or Profile depending on the output.
4. Review quality before publishing or sharing.
5. Use Membership when higher usage or deeper workflow support is needed.

## AI Workflows

Use AI for fast creative support:

- hooks
- captions
- short concepts
- text variations
- visual prompt ideas
- campaign angles
- product or content descriptions

Use Agent for structured support:

- launch steps
- content plans
- rewrite passes
- production checklists
- idea-to-action breakdowns
- next-step decisions

Always review AI output before publishing. AI can accelerate work, but the creator remains responsible for final voice, accuracy, rights, and tone.

## Content Creation

SkyOS supports creator content across profile, music, video, and shop surfaces.

Quality checks before publishing:

- title is clear
- media loads correctly
- external provider link is valid
- profile context is current
- product or order wording is accurate
- no placeholder text remains
- permissions match the intended audience

## Publishing And Assets

Asset work should be intentional.

Before uploading or changing media:

- use the correct owner/profile context
- confirm file type and size are supported
- avoid copyrighted material you cannot use
- keep source files organized outside the app
- verify the final presentation on mobile

If an upload fails, check network, file type, and permissions before retrying.

## Profile

Profile is the public or semi-public identity layer. Keep it aligned with the creator's current work.

Review:

- display name
- bio or context
- gallery content
- linked creator areas
- visible actions
- outdated references

Profile should not become a dumping ground for unfinished experiments.

## Music And Video

Use Music and Video as curated surfaces, not storage folders.

For each item:

- confirm the media opens
- confirm title and metadata
- check the provider link
- check owner/editor visibility
- remove or update stale content references

## Shop And Orders

Creator commerce should be clear and supportable.

Before promotion:

- product detail page is accurate
- price and variants are correct
- checkout or order submission path is working
- user order view shows status
- owner order management is restricted

Users should not have to guess whether an order went through.

## Membership Value

Membership should feel like expanded capability, not pressure.

Creator-facing value can include:

- higher AI usage
- richer workflow support
- stronger creator operations
- deeper agent assistance
- more consistent production flow

Upgrade messaging should be honest about what changes and what remains the same.
